chol

!THIS DOCUMENTATION IS A WORK IN PROGRESS!

Introduction

chol is a collection of header-only STB-style C libraries which include useful things like command line argument parsing, filesystem functions etc.

Usage

Since chol is an STB-style library, all you have to do is define the CHOL_IMPLEMENTATION macro and then include the header file(s) that you want to use.

You can alternatively include the header files in a .h file and then include the implementations in a .c file. Example:

example.h

#include <chol/args.h>
#include <chol/log.h>
#include <chol/fs.h>

example.c

#define CHOL_IMPLEMENTATION

#include "example.h"

Global config macros

chol has global config macros to allow for more flexibility. Some headers also have their own config macros.

CHOL_DEF

This macro is included in front of chol function definitions. On default, its defined like so:

internal/def.h

#define CHOL_DEF

You can define it before including the headers and implementations to change how the functions will be defined.

CHOL_ASSERT

This macro is used for assert statements in the chol implementations. On default, it includes assert.h and is defined like so:

internal/assert.h

#define CHOL_ASSERT(EXPR) assert(EXPR)

CHOL_ALLOC

This macro is used to allocate memory in the chol implementations. On default, it includes stdlib.h and is defined like so:

internal/alloc.h

#define CHOL_ALLOC(SIZE) malloc(SIZE)

CHOL_REALLOC

This macro is used to reallocate memory. On default, it includes stdlib.h and is defined like so:

internal/alloc.h

#define CHOL_REALLOC(PTR, SIZE) realloc(SIZE)

CHOL_FREE

This macro is used to free memory. On default, it includes stdlib.h and is defined like so:

internal/alloc.h

#define CHOL_FREE(PTR) free(PTR)
